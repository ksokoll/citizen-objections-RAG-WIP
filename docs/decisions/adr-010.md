## ADR-010: PII Masking and GDPR-Compliant Processing

**Status:** Accepted

**Context:** Einwendungen submitted by citizens may contain personal data including names, addresses, and contact information. DSGVO (GDPR) Article 5(1)(b) requires that personal data is not processed beyond the original collection purpose. The system's purpose is argument analysis and response drafting, not identity tracking.

**Decision:** PII masking is applied as the final step of DocumentIngestion, before any data is passed to downstream contexts. The raw document is stored in an access-controlled document store separate from the processing pipeline. All downstream contexts (Triage, ResponseDrafting, AuditLog) operate exclusively on the masked text. PII masking uses a combination of regex patterns for structured PII (addresses, phone numbers, email addresses) and an LLM-based pass for unstructured name detection.

**Rationale:** Masking at the ingestion boundary is the correct architectural placement because it enforces the invariant at the data flow level rather than relying on each downstream context to handle PII correctly. The raw document must be retained separately because public authorities have legal retention obligations for original submissions, but those obligations are satisfied by the access-controlled document store, not by the processing pipeline. Separating retention from processing is the DSGVO-compliant design.

**Rejected Alternatives:** Anonymization instead of masking (irreversible PII removal). Rejected because retention obligations require access to the original document under defined circumstances. Per-context PII handling without upstream masking. Rejected because it makes PII compliance dependent on every future context correctly implementing handling rules, which is not enforceable architecturally.

**Consequences:** The raw document store requires access control and a defined retention policy. The masked text is the canonical input for all processing. A test suite verifies that no PII passes the ingestion boundary under representative inputs. The masking step's accuracy is part of the Architecture Fitness Functions for this system.
