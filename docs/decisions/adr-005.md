## ADR-005: Domain-Routed Retrieval with Confidence Fallback

**Status:** Accepted

**Context:** The legal corpus covers multiple distinct domains (BauGB, VwVfG, UVPG, and potentially Landesrecht). Retrieving across the full corpus for every query introduces cross-domain noise: a BauGB objection may surface irrelevant VwVfG chunks that share surface-level vocabulary. Routing retrieval to the correct domain subset improves precision and makes the retrieval path more explainable.

**Decision:** The system uses an LLM-based classifier to identify the relevant legal domain from the extracted objection arguments before retrieval. The classifier output is used as a metadata pre-filter on the retrieval index. If classifier confidence falls below a defined threshold, the system falls back to full-corpus retrieval without filtering.

**Rationale:** This pattern is correctly named domain-routed retrieval or LLM-classified metadata pre-filtering, not multi-step retrieval. Multi-step retrieval would involve retrieving, analyzing results, and retrieving again based on those results, which is a different and more expensive pattern. The failure mode here is misclassification, not a poor retrieval iteration. The confidence fallback is the critical mitigation: a hard routing failure (classifying a BauGB objection as UVPG) with no fallback would silently return empty or irrelevant results. Full-corpus fallback degrades precision but never produces a silent failure.

**Rejected Alternatives:** Parallel retrieval across top-N domains. Rejected because it multiplies retrieval cost proportionally and complicates RRF fusion across domain-partitioned indices. Static keyword-based routing. Rejected because legal arguments frequently paraphrase domain-specific content without using exact domain identifiers.

**Consequences:** The Triage context is responsible for domain classification before handing off to the retrieval layer. The classifier confidence score and the routing decision (domain-filtered or full-corpus fallback) are logged in `retrieval_metadata` for every request. The confidence threshold is a configurable parameter included in `retrieval_config_hash`.
