# ADR-021: Exact-Match-Only Norm Resolution

## Status

Accepted.

## Context

The Retrieval bounded context (ADR-020) resolves canonical norm citations from Triage to their source Gesetzestext. The iteration_14_plan pre-registered a hybrid strategy: exact-match lookup on the paragraph-level key first, vector-similarity fallback (multilingual-e5-large embeddings over a faiss index) for granularity-drift cases such as i.V.m.-synthesised inner citations or sub-paragraph citations.

The hybrid design rested on two pre-registered hypotheses:

- H-A: exact-match alone resolves 80 to 90% of canonical citations.
- H-B: the vector fallback recovers the remainder, lifting overall recall to at least 95%.

Both were tested against the Phase A ground truth before committing to the hybrid design in production.

## Decision

Production norm resolution uses exact-match only. The vector-similarity fallback is removed from the production path.

An unresolved citation returns `NormWithSource(resolved=False)` rather than a weak vector match. Downstream (ResponseDrafting) degrades the affected argument accordingly.

## Rationale

Measured against the 25 unique must_retrieve citations in the Phase A ground truth (eval_retrieval_recall):

| Method | Count | Share |
|--------|-------|-------|
| exact | 25 / 25 | 100.0% |
| vector | 0 / 25 | 0.0% |
| unresolved | 0 / 25 | 0.0% |

Per-Gesetz, all six statutes represented in the test set (BNatSchG, BauGB, BauNVO, EnWG, WHG, WPG) resolved entirely by exact-match. Overall resolution recall was 100%.

The vector fallback performed zero real work. The paragraph-level normalisation in the exact-match path already absorbs the granularity drift that the fallback was designed to catch: a citation such as "§ 9 Abs. 1 Nr. 1 WHG" is reduced to the key "§ 9 WHG", which is present in the index. Because the norm_extractor (Iteration 9, Iteration 12) and the XML loader both produce clean canonical "§ N Gesetz" forms, every valid citation hits an exact key.

The fallback is not merely unused; it is actively harmful when it does fire. The setup smoke test resolved the non-existent "§ 999 WHG" to "§ 105 WHG" with a cosine score of 0.801, a confident-looking but wrong match. In a Behörde context, returning the wrong Gesetzestext is worse than an honest "not found": a Sachbearbeiter shown a plausible but incorrect statute passage may rely on it. Exact-match-only fails honestly, returning `resolved=False` for a citation it cannot key.

Removing the fallback also removes the multilingual-e5-large dependency (a 2 GB model and a multi-minute CPU index build), the faiss-cpu dependency, and the need to calibrate a confidence floor. The resolver becomes a dictionary lookup: sub-millisecond, deterministic, no external model.

This matches the architecture decision hierarchy: the simplest mechanism that solves the problem, lowest complexity, lowest operational burden, and the decision is cheaply reversible.

## Consequences

Positive:

- Production resolution is a dictionary lookup. Sub-millisecond, deterministic, no embedding model, no vector index, no network, no model download.
- No risk of a confident-but-wrong vector match presenting an incorrect Gesetzestext to a Sachbearbeiter.
- The dependency surface shrinks: sentence-transformers and faiss-cpu leave the production path.
- Unit tests for the resolver need no model and no index fixture.

Negative:

- A citation whose paragraph key is absent from the corpus resolves to `resolved=False`. There is no semantic recovery attempt. This is the intended behaviour (honest miss over confident wrong match), but it means malformed or out-of-corpus citations are not rescued.
- The conclusion rests on a 25-citation test set. It is robust on the observed data but not statistically saturated. If production data later exhibits genuine drift that exact-match misses, this decision is revisited (see reversibility note).

## Reversibility

The decision is deliberately reversible. The vector-fallback implementation (E5Embedder, FaissNormIndex, the hybrid resolution path) is retained as experimental reference code under `experiments/`, not deleted. If production data later shows citations that exact-match cannot resolve, the hybrid path can be reinstated with a confidence floor calibrated against that data. The floor would need to sit well above 0.801 given the observed false-positive score; the calibration data does not yet exist because no real citation has needed the fallback.

## Alternatives Considered

A. Keep the hybrid path with the floor raised (for example to 0.93). Rejected. It retains the full e5 and faiss complexity, the model download, and the index build, to guard against a drift case that did not occur in the test set. The cost is paid on every run; the benefit is hypothetical.

B. Keep the hybrid path at the original 0.80 floor. Rejected. The smoke test demonstrated this floor admits confident-wrong matches (§ 999 WHG to § 105 WHG at 0.801). In a legal context this is the worst failure mode.

C. Exact-match only, with the vector path retained as reversible experimental reference. Chosen. Matches the measured data, removes the harmful failure mode, and keeps the option open without paying for it in production.
